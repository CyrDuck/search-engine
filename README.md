# XJCO3011 Coursework 2 — Search Engine Tool

A Python search engine that crawls [quotes.toscrape.com](https://quotes.toscrape.com/), builds an inverted index with **TF-IDF ranking**, and provides an interactive command-line interface for querying.

---

## Features

- **Polite crawling** — 6-second politeness window between requests, with retry logic and graceful error handling
- **Inverted index** — stores term frequency, word positions, and TF-IDF scores per document
- **TF-IDF ranking** — results sorted by relevance, not just presence
- **Multi-word queries** — AND semantics: returns pages containing *all* query terms
- **Spelling suggestions** — "Did you mean…?" via Ratcliff/Obershelp similarity
- **Persistent storage** — human-readable JSON index; save once, query many times
- **Complexity analysis** — built-in `stats` command reports index density and algorithmic complexity

---

## Architecture

```
src/
├── crawler.py   — BFS web crawler with politeness, retry, and HTML parsing
├── indexer.py   — Inverted index, TF-IDF, JSON persistence
├── search.py    — Query interface, formatted output, spelling suggestions
└── main.py      — Interactive CLI shell (REPL)

tests/
├── test_crawler.py  — Unit + integration + performance tests for crawler
├── test_indexer.py  — Unit + integration + performance tests for indexer
└── test_search.py   — Unit + integration + edge-case tests for search

data/
└── index.json   — Compiled index (generated by `build` command)
```

### Data Structure

The core data structure is a nested `dict`:

```python
index: dict[str, dict[str, WordStats]]
#       term      url       statistics
```

| Design choice | Rationale |
|---|---|
| Hash map (dict) | O(1) average-case lookup by term and URL |
| JSON persistence | Human-readable, inspectable without running the tool |
| TF-IDF scoring | Industry-standard relevance signal; penalises common terms |
| Stop-word filtering | Reduces index size; improves retrieval precision |
| Position storage | Enables future phrase-proximity ranking |

**Complexity summary:**

| Operation | Time | Space |
|---|---|---|
| Index build | O(W) — W = total words | O(U × D) — U terms, D docs |
| TF-IDF compute | O(U × D) | O(1) extra |
| Lookup (get_postings) | O(1) average | — |
| find() | O(Q × D log D) | O(D) |
| save / load | O(U × D) | O(U × D) |

## Usage

Start the interactive shell:

```bash
python src/main.py
```

### Commands

```
> build
```
Crawls `https://quotes.toscrape.com/` (observing a 6-second politeness window), builds the inverted index, and saves it to `data/index.json`.  
*Takes several minutes to complete the full crawl.*

```
> load
```
Loads a previously built index from `data/index.json`. Run this on subsequent sessions to avoid re-crawling.

```
> print nonsense
```
Prints the full inverted index entry for the word `nonsense`, including URL, title, frequency, TF-IDF score, and word positions.

```
> find indifference
> find good friends
```
Returns all pages containing the query term(s), ranked by TF-IDF score. Multi-word queries use AND semantics (all terms must be present).

```
> stats
```
Displays index statistics and a complexity analysis report.

```
> help
```
Lists all available commands.

## Testing

Run the full test suite with coverage:

```bash
pytest tests/ -v --cov=src --cov-report=term-missing
```

Run a specific test file:

```bash
pytest tests/test_indexer.py -v
```

### Test strategy

| Layer | File | What's tested |
|---|---|---|
| Unit | `test_crawler.py` | URL normalisation, domain filtering, HTML parsing, HTTP errors, retries |
| Unit | `test_indexer.py` | Tokenisation, stop words, frequency counts, TF-IDF correctness, round-trip persistence |
| Unit | `test_search.py` | All four command handlers, edge cases, spelling suggestions |
| Integration | All files | Multi-page indexing, save→load→find workflow |
| Performance | All files | Benchmark: 1000 pages indexed < 5s; 100 find() calls < 1s |
